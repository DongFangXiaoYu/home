Linux 启动脚本

Linux在启动过程中会执行几个文本文件中的命令. 包含Linux命令的文本文件称为启动脚本.

修改启动脚本可以再服务器启动时候设置Java环境变量, 达到配置Java环境的目的:

  1. 修改启动脚本

    su
cd /etc 
cp profile profile.2018.3.1
vim profile

    修改启动脚本之前务必进行备份, 用于意外恢复之用

  2. 启动脚本添加内容:

    export JAVA_HOME=/usr/local/java/jdk1.8.0_161
export CLASSPATH=.
export PATH=/usr/local/java/jdk1.8.0_161/bin:$PATH

  3. 对启动脚本进行测试

    source profile
javac -version
java -version

  4. 如果没有问题, 则可以重新启动Linux

    reboot

安装Tomcat

利用zip安装tomcat

  1. https://tomcat.apache.org, 下载tomcat
  2. unzip apache-tomcat-7.0.85.zip
  3. cd apache-tomcat-7.0.85/bin
  4. chmod +x *.sh //增加执行权限
  5. 启动 ./startup.sh
  6. 停止 ./shutdown.sh

如果没有unzip命令, 则可以使用 yum install unzip 安装unzip命令

利用 yum 命令安装

  1. yum install tomcat
  2. 启动 systemctl start tomcat.service
  3. 停止 systemctl stop tomcat.service
  4. 重新启动 systemctl restart tomcat.service

  5. Tomcat的文件夹:

    webapps: /var/lib/tomcat/webapps
conf: /etc/tomcat

yum安装的Tomcat没有默认页面(Tom猫), 可以将将zip中的ROOT文件夹复制过来:

cp -r ~/apache-tomcat-7.0.85/webapps/ROOT /var/lib/tomcat/webapps

安装后可以利用 ip:8080 访问Tomcat

安装MySQL

Oracle 拥有MySQL, 开源组织分支了: MariaDB, 在yum仓库中的数据库不是MySQL而是MySQL的分支版本 MariaDB.

安装MySQL(MariaDB):

yum -y install mariadb mariadb-server

一行命令安装了两个组件, mariadb 是数据库的客户端, mariadb-server是数据库的服务器端.

启动/关闭MySQL服务器:

systemctl start mariadb.service
systemctl stop mariadb.service

配置数据库文字编码, 修改 /etc/my.cnf 

[mysqld]
character-set-server=utf8

配置编码以后需要重新启动服务器.

重新启动服务器:

systemctl restart mariadb.service

检查数据库编码, 用客户端连接到MySQL,执行:

show variables like 'char%';

如果没有是正确的编码则显示结果是:
